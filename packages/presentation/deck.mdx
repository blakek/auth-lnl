## What do they mean?

> **Authentication** is the act of confirming the truth of an attribute of a
> single piece of data claimed true by an entity.

> **Authorization** is the function of specifying access rights/privileges to
> resources related to information security and computer security in general and
> to access control in particular.

## Common Usages in Web Services

- Basic Auth

---

## Basic Auth

- oldest standard in this talk
- easy to implement
- needs HTTPS to be secure

---

## OAuth 2.0

- flexible <And /> secure
- easier for admins
- sometimes confusing terminology
- deals with authorization only
- can “add on” authentication; OpenID Connect

---

## SAML

- notoriously complicated
- easier for admins
- specs for authentication and authorization
- “Enterprise” SSO
